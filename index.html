Структура репозитория

Файловая структура

  • positions/universal/ — базовое резюме (HTML + PDF), основа для адаптаций
  • positions/[slug]/ — адаптации под конкретные вакансии (создаются по запросу, удаляются после завершения цикла)
  • shared/ — фото, мастер-профиль, общие ресурсы
  • shared/master_profile_ru.md — мастер-профиль v1.7.7, источник истины для всех адаптаций
Правила адаптаций

Принципы создания позиций

  • База — всегда universal/index.html, цвета адаптируются под корпоративный стиль работодателя
  • Заголовок роли — универсальный, не переписывается под текст вакансии в лоб
  • HTML-итерации первые, PDF только после согласования вёрстки
  • Cover letter — только после явного запроса, после утверждения PDF
  • Метрики и цифры — только из мастер-профиля, без подгонки под вакансию
  • Завершённые формы глаголов: «Спроектировал», «Обеспечил», «Вывел»
  • Зарплата: от 500 000 руб. (в отдельных позициях — без «от»)
  • Номера страниц PDF: формат «N из M»

Что не делать

  • Co-Founder — убран из всех резюме
  • LoRA — не упоминать в стеке
  • «цифровые экосистемы» — заменять на «цифровое пространство» / «цифровая платформа»
  • ROI 1,57× — не упоминать в резюме
  • «Резюме онлайн» — не добавлять в блок контактов
  • GigaChat / YandexGPT — не упоминать в связке с закрытым госконтуром
